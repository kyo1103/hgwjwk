# hgwjwk 협업 규칙

이 저장소는 `kyo`와 세무사님 1명이 같이 작업하는 것을 기준으로 운영합니다.

## 기본 원칙

- `master`는 최종 확인된 내용만 올립니다.
- 각자 작업은 본인 브랜치에서 합니다.
- 커밋은 짧게, 자주 합니다.
- 작업 후 바로 `push`해서 GitHub에 올립니다.
- 다른 사람이 만든 코드는 먼저 `pull`로 최신 상태를 받고 작업합니다.

## 브랜치 규칙

- `kyo` 작업 브랜치 예시: `kyo/work`
- 세무사님 작업 브랜치 예시: `tax/work`

처음 한 번만 만들면 됩니다.

```bash
git checkout -b kyo/work
git push -u origin kyo/work
```

## 작업 순서

항상 아래 순서로 작업합니다.

```bash
git pull
git add .
git commit -m "작업 내용"
git push
```

## 커밋 메시지 규칙

- `feat: 새 기능 추가`
- `fix: 오류 수정`
- `docs: 문서 수정`
- `style: 문구/디자인 수정`

예시:

```bash
git commit -m "feat: 고객사 요청 화면 수정"
git commit -m "fix: 세금 파일 업로드 오류 수정"
```

## 충돌 방지 규칙

- 같은 파일을 동시에 크게 수정하지 않습니다.
- 작업 시작 전 `git pull`을 먼저 합니다.
- 큰 수정 전에 상대방에게 먼저 말합니다.
- `master`에 직접 작업하지 않습니다.

## 간단 운영 규칙

- 화면/기능 수정은 `kyo`가 우선 작업합니다.
- 세무 기준, 문구, 정책 확인은 세무사님이 확인합니다.
- 확인이 끝난 내용만 `master`에 반영합니다.
