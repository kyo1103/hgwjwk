Internal Operations

운영 콘솔은
업무 우선순위를 먼저 보여줘야 합니다.

현재 워크스페이스는 자문 운영, 고객 포털, 자동화 ERP로 나뉘어 있습니다. 내부 운영 허브에서는 요청과 고객사, 로그와 ERP 상태를 같은 운영 시선으로 연결해 주는 것이 핵심입니다.

Current Surfaces

OPS바로가기

운영 콘솔

노무·세무 담당자가 요청, 태스크, 보고, 템플릿, 고객사를 운영하는 내부 업무 허브입니다.

PORTAL바로가기

고객 포털

고객사는 인사 자료, 계약, 휴가, 파일, 문의 내역을 한 화면에서 확인합니다.

ERP바로가기

자동화 ERP

정부 포털 자동화, 증빙 수집, 작업 재시도, 결과 보관을 담당하는 수집 허브입니다.

INBOX

업무 인박스

서비스 요청, 후속 업무, 담당자 할당을 기준으로 자문 운영을 통합 관리합니다.

ERP

자동화 ERP

홈택스·4대보험·Gov24·위택스 수집 작업과 문서 결과를 별도 콘솔에서 관리합니다.

TENANT

고객사 관리

고객사별 계약 상태, 소속 사용자, 서비스 범위, 문서 흐름을 관리합니다.

AUDIT

운영 로그

요청 처리 이력과 시스템 로그를 함께 보고, 장애나 누락을 빠르게 추적합니다.

Trust Checklist

공동인증서 최소 노출

p12는 저장 시 즉시 암호화하고, 워커 실행 직전에만 복호화한 뒤 임시 경로를 바로 정리합니다.

테넌트 강제 스코프

Firm 또는 Tenant 키를 모든 조회와 변경 경로에 주입해 다른 사무소 데이터 접근을 차단합니다.

행위 기반 감사 로그

요청 생성, 문서 발급, 실패 재시도, 전송 이력을 같은 로그 모델에서 추적합니다.

운영자 보안 기본값

2FA, IP 로깅, HTTPS, 비밀키 외부 관리가 제품 기본값이어야 합니다.

Delivery Sequence

Phase 1
1~2개월

테넌트·고객사·기본 UI

고객사 CRUD, 역할 분리, 요청/태스크 뼈대를 먼저 안정화합니다.

Phase 2
2~3개월

홈택스 문서 자동화

공동인증서 기반 로그인과 민원서류 PDF 발급 흐름을 고정합니다.

Phase 3
3~4개월

4대보험 고지 조회

건강보험·국민연금 데이터를 월별로 적재하고 엑셀/리포트로 연결합니다.

Phase 4
4~5개월

카카오 급여 요청

급여자료 요청, 고객 회신, 처리 상태를 월 단위 운영 흐름으로 완성합니다.
